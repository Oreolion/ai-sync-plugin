# Contributing to ai-sync

Thanks for your interest in contributing to ai-sync! This project enables seamless handoff between AI coding agents, and contributions from the community make it better for everyone.

## Project Structure


## How It Works

This is a **Claude Code plugin** — there is no compiled code. Commands are Markdown files with YAML frontmatter that Claude Code interprets as instructions. The skill is an auto-loaded Markdown document. Hooks are JSON-configured prompts.

## Development Setup

```bash
# Clone the repo
git clone https://github.com/Oreolion/ai-sync.git
cd ai-sync

# Load the plugin in Claude Code
claude --plugin-dir .

# Test commands in any project
/sync-init
/sync-status
/handoff
/sync-resume
/sync-diff
/sync-adapter cursor
```

## Running Tests

```bash
bash tests/validate-plugin.sh
```

The validation script checks plugin structure, JSON validity, frontmatter consistency, and cross-references between files.

## How to Contribute

### Adding a New Adapter

Adapters are thin config files that point other AI tools to `.ai-sync/`. To add support for a new tool:

1. Research the tool's configuration file format (e.g., `.cursorrules` for Cursor)
2. Add the adapter template to `/sync-adapter` command in `commands/sync-adapter.md`
3. Add the tool to the supported tools table in `README.md`
4. Update `commands/sync-init.md` if the adapter should be generated by default
5. Open a PR with the tool name in the title (e.g., "Add adapter: ToolName")

### Adding a New Command

Commands live in `commands/` as Markdown files with YAML frontmatter:

```yaml
---
description: One-line description shown in command list
allowed-tools: [Read, Write, Edit, Glob, Grep]
---

# Command Title

## Instructions

1. Step one...
2. Step two...
```

Guidelines:
- Keep `allowed-tools` minimal — only grant tools the command actually needs
- Read-only commands should not include Write or Edit
- Include clear step-by-step instructions
- Add the command to `skills/ai-sync-protocol/SKILL.md` and `README.md`

### Modifying the Protocol

The `.ai-sync/` protocol (file formats for HANDOFF.md, PROGRESS.md, PLAN.md) is defined in `skills/ai-sync-protocol/SKILL.md`. This is the single source of truth.

Changes to the protocol must be backward-compatible. An agent reading old-format files must still function.

### Pull Request Process

1. Fork the repo and create a branch from `main`
2. Make your changes
3. Run `bash tests/validate-plugin.sh` and ensure it passes
4. Update documentation if you changed commands or the protocol
5. Open a PR with a clear description of what changed and why

### Reporting Issues

Use the issue templates:
- **Bug Report** — something isn't working as expected
- **Feature Request** — suggest a new capability
- **New Adapter Request** — request support for an AI tool

## Code of Conduct

Be respectful, constructive, and inclusive. We're all here to make AI agent collaboration better.

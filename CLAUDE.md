# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

**ai-sync** is a Claude Code plugin that enables cross-platform AI agent synchronization. It maintains a `.ai-sync/` directory in target projects so agents (Claude Code, Codex, Cursor, Aider, Cline, Windsurf, OpenCode, Continue.dev, GitHub Copilot) can hand off work seamlessly via standardized Markdown files.

This is **not** a compiled project ‚Äî there is no build system, no runtime dependencies. The entire plugin is Markdown + YAML + JSON, following the Claude Code Plugin Spec v1.

## Development

```bash
# Run the plugin locally during development
claude --plugin-dir .

# Test commands in any project
/sync-init              # Bootstrap .ai-sync/ in a project
/handoff                # Capture state before switching agents
/sync-status            # Show progress and next steps
/sync-resume            # Resume from last handoff with conflict detection
/sync-diff              # Show changes since last handoff
/sync-adapter cursor    # Generate adapter for a specific tool

# Run plugin validation tests
bash tests/validate-plugin.sh
```

There are no build, lint, or test commands for this repo itself beyond `validate-plugin.sh`. Verification commands (`tsc --noEmit`, `npm run build`) run inside *target projects* during `/handoff`, not in this plugin repo.

## Architecture

The plugin has four components, all declared via the manifest at `.claude-plugin/plugin.json`:

### Commands (`commands/`)
Markdown files with YAML frontmatter defining slash commands. Each specifies `description`, `allowed-tools`, and step-by-step instructions that Claude Code follows when the user invokes the command.

- `sync-init.md` ‚Äî Creates `.ai-sync/` directory + adapter files for all supported tools
- `handoff.md` ‚Äî Conflict detection, auto-update PROGRESS.md, writes HANDOFF.md/session log
- `sync-status.md` ‚Äî Read-only status display (only allows Read, Glob, Grep)
- `sync-resume.md` ‚Äî Loads full handoff context, records conflict baseline, begins next steps
- `sync-diff.md` ‚Äî Shows committed + uncommitted + untracked changes since handoff timestamp
- `sync-adapter.md` ‚Äî Generates adapter files on demand for any supported tool

### Skill (`skills/ai-sync-protocol/SKILL.md`)
Auto-loaded when a project has `.ai-sync/` or the user mentions handoff-related keywords. Contains the full synchronization protocol spec including HANDOFF.md YAML frontmatter format, PROGRESS.md checklist format, conflict detection, auto-progress tracking, and the five key protocol rules.

### Hooks (`hooks/hooks.json`)
Two prompt-based hooks:
- **SessionStart** ‚Äî Silently checks for `.ai-sync/HANDOFF.md` and loads context
- **Stop** ‚Äî Reminds user to run `/handoff` if meaningful changes were made

### Adapter Files (generated by `/sync-init` and `/sync-adapter`)
Thin pointer files that tell each AI tool to read `.ai-sync/`: `AGENTS.md`, `.cursorrules`, `.clinerules`, `.windsurfrules`, `.aider.conf.yml`, `.github/copilot-instructions.md`, `.continuerules`. Adapter formats are defined in `commands/sync-adapter.md` as the single source of truth.

## Protocol File Formats

HANDOFF.md uses YAML frontmatter (`last_agent`, `timestamp`, `status`, `current_phase`, `current_task`, `stop_reason`) followed by sections: What Was Completed, Work In Progress, Next Steps, Files Modified/Created, Blockers, Key Decisions, Build Status.

PROGRESS.md uses `## Phase N: Name ‚Äî COMPLETE|IN PROGRESS|PENDING` headers with `- [x]`/`- [ ]` task checklists.

## Editing Conventions

- Command files must keep the YAML frontmatter `allowed-tools` array minimal ‚Äî only grant tools the command actually needs
- Read-only commands (sync-status, sync-diff) should not include Write or Edit in allowed-tools
- The skill's `description` field in frontmatter is the trigger ‚Äî it must list all relevant keywords for auto-loading
- Hook prompts should be terse (one-line reminders, silent reads)
- All protocol file formats are defined in `skills/ai-sync-protocol/SKILL.md` ‚Äî keep that as the single source of truth
- Adapter template content is defined in `commands/sync-adapter.md` ‚Äî update there, not in sync-init.md
- After making changes, run `bash tests/validate-plugin.sh` to verify plugin integrity
